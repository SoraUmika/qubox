A comprehensive sweep of the codebase was performed to eliminate all stale
imports, broken constructor calls, naming inconsistencies, and documentation
inaccuracies.  All changes below have been verified with import tests.

\subsection{Phase 1: SessionManager Runtime Fixes}

\texttt{experiments/session.py} had \textbf{6 broken call sites} that would
crash at runtime.  Each was corrected to match the actual v3 constructor
signatures:

\begin{longtable}{p{5cm}p{4cm}p{5cm}}
\toprule
\textbf{Call Site} & \textbf{Bug} & \textbf{Fix} \\
\midrule
\endhead
\texttt{ConfigEngine()} &
  \texttt{hardware\_json=}, \texttt{pulse\_json=} (no such params) &
  \texttt{hardware\_path=} (single path) \\
\texttt{QueueManager()} &
  \texttt{controller=self.hardware} &
  \texttt{runner=self.runner} \\

  \texttt{POM(self.config\_engine)} &

\subsection{Phase 6: External LO Detection Fix}

\texttt{hardware/controller.py} methods \texttt{\_element\_octave\_rf\_out()}
and \texttt{\_is\_external\_lo()} were using \texttt{self.qm.get\_config()}
which returns the SDK-normalized config where \texttt{RF\_inputs} is stripped.
This caused \texttt{UnableToSetFrequencyError} for elements connected to
external LO sources.

\textbf{Fix:} Both methods now use \texttt{self.config.hardware\_base} (the
original un-normalized config) to correctly detect external LO routing.

  \item \texttt{from ..gates\_legacy import QubitRotation}
